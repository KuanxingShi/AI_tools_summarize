# 全局配置

## 语言规范（强制执行）

无论用户使用何种语言提问，所有回复（包括解释、计划、总结等）必须使用中文。

- **所有对话和文档**都使用中文
- **注释**优先使用中文，除非项目已有明确的英文注释规范
- **错误提示**使用中文
- **文档**使用中文 Markdown 格式

## 代码修改痕迹（强制执行）

**触发条件**：每次代码修改

**强制执行动作**：

- 必须在代码修改处留下清晰的注释，标记该变更为 AI 生成
- 内容需包含：修改时间、修改者（默认为 kxshi）、修改需求简述
- 格式：`// [AI] <日期> <修改者>: <修改需求>`
- 示例：`// [AI] 2023-10-27 kxshi: 优化文档解析逻辑`

## Java 代码规范

**命名规范**：

- 类名使用 PascalCase
- 方法和变量名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

**格式化**：

- 保持代码缩进、空格和换行的一致性，遵循项目的格式化配置

**注释**：

- 关键逻辑、复杂算法和公共接口需添加 JavaDoc 或行内注释

**异常处理**：

- 合理捕获和处理异常，避免空的 catch 块

**代码简洁性**：

- 避免冗余代码，保持方法简短且单一职责
